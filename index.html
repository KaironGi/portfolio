Projetos

Controle de Estoque

Sobre

Este projeto é um Sistema Completo de Gestão de Estoque desenvolvido para oferecer controle total sobre produtos, clientes e movimentações comerciais. A aplicação foi construída com foco em organização, rastreabilidade e eficiência operacional, integrando múltiplos módulos que trabalham de forma sincronizada.

O sistema permite gerenciar desde o cadastro de produtos até o acompanhamento detalhado de compras e trocas, garantindo uma visão clara e estratégica do estoque.

Principais funcionalidades

Cadastro de Clientes

- Registro completo de clientes

- Organização de dados para controle de compras

- Integração histórico de pedidos

Cadastro de Produtos

- Nome, preço e quantidade em estoque

- Informações adicionais e observações

- Atualização automática de estoque após vendas e trocas

Sistema de Sacola de Compras

- Adição de múltiplos produtos

- Cálculo automático do total

- Campo de observação do pedidos

- Opção de marcar compra como presente

- Finalização integrada ao histórico

Histórico de Compras

- Registro completo de todas as vendas

- Rastreabilidade das movimentações

- Consulta organizada por clientes

Sistema de Trocas

- Troca de produtos com ajuste automático de estoque

- Registro no histórico para manter integridade das Informações

- Controle seguro das movimentações

DashBoard e Alertas de Estoque Baixo

- Identificação automática de produtos com baixo estoque

- Alertas visuais para facilitar a tomada de decisão

- Prevenção de ruptura de estoque

O objetivo do sistema é oferecer uma solução prática e eficiente para gestão de estoque e vendas, permitindo um controle estratégico das operações comerciais e garantindo maior organização e segurança nos processos.

Promissum Bank

Sobre

O Promissum Bank é um sistema bancário modular desenvolvido com foco em regras de negócios reais do setor financeiro, como controle de transações, auditoria risco e consistência de dados. O projeto simula a estrutura de um core bancário simplificado, priorizando arquitetura, segurança e integridade das operações.

O objetivo não foi apenas "fazer funcionar", mas construir algo próximo do que é exigido em ambientes financeiros de alta responsabilidade.

Arquitetura e Tecnologias

- Java (orintação a objetos e modelagem de domínio)

- SQLite3 para banco de dados

Estrutura modular separando:

- Domínio (regras de negócio)

- Serviços

- Camada de persistência

- Controle de concorrência em operações críticas

- Estrutura preparada para futura evolução para API REST

Principais Funcionalidades

- Abertura de contas com status (ativa, bloqueada, encerrada)

- Depósito, saque e transferência entre contas

- Registro de transações para auditoria

- Preparação para rollback de operações

- Estrutura pensada para evitar inconsistência de saldo
